# Global Claude Settings

## 🔑 환경변수 및 보안

민감정보는 항상 `~/.claude/.env`에서만 관리한다:
- `GITHUB_TOKEN`: GitHub MCP 및 gh CLI 인증
- `NOTION_TOKEN`: Notion MCP 인증

보안 규칙:
- `.env`, `*.pem`, `*.key`, `secrets.*` 파일은 절대 커밋하지 않는다
- push/commit 전 반드시 민감파일 staged 여부를 확인한다
- 토큰/시크릿을 커밋 메시지나 코드에 하드코딩하지 않는다

---

## 🧠 맥락 추론 규칙 (모든 커맨드 공통)

owner / repo / branch 등 필수 정보가 명시되지 않은 경우:
1. $ARGUMENTS에서 직접 추출
2. 이번 대화의 이전 메시지에서 언급된 값 참조
3. 로컬 git 상태 또는 MCP로 원격 상태 조회
4. 위 모두 불명확할 때만 사용자에게 질문

불필요한 질문을 최소화하고, 맥락을 우선 활용한다.

## 📐 스킬 / 커맨드 / 에이전트 작성 Best Practice

> 스킬, 커맨드(.claude/commands/), 에이전트(.claude/agents/)를 새로 만들거나
> 수정할 때는 반드시 아래 원칙을 따른다.
> 출처: https://code.claude.com/docs/en/best-practices

### 1. 검증 수단을 항상 포함한다
커맨드/스킬이 작업을 수행한 후 결과를 스스로 검증할 수 있도록 구성한다:
- 코드 작성 → 테스트 실행
- UI 변경 → 스크린샷 비교
- 빌드/배포 → 성공 여부 확인
- "그럴듯해 보이는" 결과가 아닌, 실제로 동작함을 확인하는 단계를 넣는다

### 2. 탐색 → 계획 → 실행 순서를 지킨다
복잡한 커맨드일수록 이 순서를 명시한다:
- **탐색**: 관련 파일/코드/맥락 파악 (변경 없음)
- **계획**: 구체적인 실행 계획 수립
- **실행**: 계획 기반으로 구현
- **검증**: 결과 확인
작은 단일 작업(오타 수정, 변수명 변경 등)은 계획 없이 바로 실행해도 된다.

### 3. 구체적인 컨텍스트를 제공한다
커맨드 안에서 Claude가 참조할 파일, 패턴, 제약사항을 명시한다:
- 모호한 지시 대신 구체적인 파일/경로/패턴을 가리킨다
- 기존 코드베이스의 패턴을 참고하도록 안내한다
- 증상과 예상 결과를 함께 제공한다

### 4. CLAUDE.md vs 스킬 구분 원칙
- **CLAUDE.md**: 모든 세션에서 항상 적용되는 전역 규칙 (짧고 핵심만)
- **스킬**: 특정 상황에서만 필요한 도메인 지식/워크플로우
- CLAUDE.md가 너무 길어지면 Claude가 규칙을 무시한다 → 주기적으로 정리한다
- 각 줄마다 "이게 없으면 실수를 할까?" 자문하고, 아니면 삭제한다

### 5. 커맨드 frontmatter 필수 항목
```
---
description: (필수) 한 줄 설명 — 자동완성 및 Claude 자동탐지에 사용됨
argument-hint: (권장) 인자 힌트 e.g. [topic], [issue-number]
allowed-tools: (필요한 툴만) 최소 권한 원칙 적용
model: (선택) haiku(빠름/단순) | sonnet(기본) | opus(복잡)
---
```

### 6. 에이전트 작성 원칙
에이전트는 독립된 컨텍스트에서 실행되므로:
- 역할과 전문 영역을 명확히 정의한다
- 필요한 툴만 허용한다 (최소 권한)
- 메인 대화 컨텍스트를 오염시키지 않는 조사/검토 작업에 활용한다
- 에이전트 결과는 요약해서 메인 컨텍스트로 반환한다

### 7. 컨텍스트 관리
- 커맨드가 대량의 파일을 읽어야 하면 서브에이전트로 위임한다
- 조사(investigation)와 구현(implementation)을 같은 커맨드에 합치지 않는다
- 무거운 탐색은 `use subagents to investigate`로 분리한다

### 8. 흔한 실수 패턴 (피해야 할 것)
- ❌ 검증 단계 없이 구현만 하는 커맨드
- ❌ 너무 많은 작업을 하나의 커맨드에 몰아넣기
- ❌ 경로/파일명을 하드코딩 (맥락 추론 또는 $ARGUMENTS 활용)
- ❌ allowed-tools를 너무 넓게 설정 (`*` 남용)
- ❌ 민감파일 체크 없이 push/commit 실행

## 📁 커맨드 위치 및 우선순위

| 위치 | 스코프 | 우선순위 |
|---|---|---|

| `.claude/commands/` | 해당 프로젝트만 | 1순위 (덮어씀) |
| `.claude/agents/` | 서브에이전트 정의 | — |

- `fix:` 버그 수정
- `docs:` 문서
- `refactor:` 리팩토링
- `chore:` 기타 유지보수
