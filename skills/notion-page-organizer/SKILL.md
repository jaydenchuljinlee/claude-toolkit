# Notion Page Organizer

노션의 특정 페이지 아래에 흩어져 있는 하위 페이지들을 분석하여, 주제별 카테고리 폴더로 자동 분류·정리하는 스킬. 3단계 이상의 깊이도 지원한다.

## 전제 조건

- Notion MCP 연결이 활성화되어 있어야 한다.
- 사용 도구: `notion-fetch`, `notion-search`, `notion-create-pages`, `notion-move-pages`

---

## 워크플로우

### Step 1: 대상 페이지 확인

사용자에게 정리할 상위 페이지를 확인한다.

- 사용자가 페이지 URL이나 이름을 제공한 경우 → 바로 진행
- 제공하지 않은 경우 → `notion-search`로 후보를 찾아 확인 요청

### Step 2: 페이지 구조 파악

`notion-fetch`로 대상 페이지의 전체 구조를 가져온다.

확인할 것:
- 모든 하위 페이지의 제목과 ID
- 이미 카테고리 역할을 하는 하위 페이지가 있는지 (하위에 또 페이지를 가진 페이지)
- 하위 페이지 안에 또 하위 페이지가 있는지 (깊이 파악)
- 데이터베이스가 섞여 있는지

필요하다면 하위 페이지를 재귀적으로 fetch하여 전체 트리를 파악한다.

### Step 3: 자동 카테고리 분류

수집한 하위 페이지 제목들을 분석하여 카테고리 트리를 구성한다.

#### 분류 원칙

1. **제목 기반 자동 분류**: 페이지 제목의 주제를 파악하여 적절한 카테고리에 배치한다.
2. **기존 카테고리 재활용**: 이미 카테고리 역할을 하는 폴더가 있으면 그대로 활용한다.
3. **새 카테고리 자동 생성**: 기존 카테고리에 맞지 않는 주제는 새 카테고리를 만든다.
4. **3단계 깊이 지원**: 대분류 → 중분류 → 페이지 구조를 적극 활용한다.
   - 예: `백엔드/ORM/JPA 사용관련`, `백엔드/ORM/Hibernate`
   - 단, 중분류에 페이지가 1~2개뿐이면 굳이 3단계로 쪼개지 않는다.
5. **한 페이지 = 한 카테고리**: 하나의 페이지는 가장 적합한 하나의 곳에만 넣는다.
6. **카테고리 최소 기준**: 카테고리에 페이지가 1개뿐이어도, 향후 확장 가능성이 높은 주제라면 독립 카테고리로 유지한다.
7. **아이콘 자동 부여**: 카테고리 주제에 맞는 이모지 아이콘을 자동으로 선택한다. 사용자에게 아이콘을 묻지 않고, 주제에 맞게 직접 결정한다.

#### 3단계 분류 예시

```
개인공부/
├── 🖥️ 백엔드/
│   ├── 📦 ORM/
│   │   ├── JPA 사용관련
│   │   └── Hibernate
│   ├── 🗄️ 데이터베이스/
│   │   ├── MySQL
│   │   └── Redis
│   ├── DDD
│   └── Spring Security
├── 🤖 AI & 머신러닝/
│   ├── LLM
│   └── RAG
├── 💻 프로그래밍/
│   ├── 🔄 비동기/
│   │   ├── coroutine
│   │   └── Reactor
│   └── 알고리즘
├── 🔧 임베디드/
│   └── 차량 제어 시스템
├── ☁️ 인프라/
│   └── AWS
└── 🌍 외국어/
    ├── 🇯🇵 일본어
    └── 🇬🇧 영어
```

#### 깊이 판단 기준

- **2단계로 충분**: 한 카테고리에 5개 이하이고 주제가 유사한 페이지들
- **3단계 필요**: 한 카테고리에 6개 이상이거나, 내부에 뚜렷한 하위 그룹이 존재
- **3단계 이상**: 하위 그룹이 또다시 세분화 가능한 경우 (드물지만 지원)

### Step 4: 분류 계획 제시 및 확인 (필수)

분류 결과를 트리 형태로 사용자에게 보여주고 **반드시** 확인을 받는다.

표시할 정보:
- 전체 트리 구조 (깊이 포함)
- (새 카테고리) 표시로 새로 생성할 폴더 명시
- 새로 생성할 카테고리 수, 이동할 페이지 수 요약
- 분류가 애매한 페이지가 있으면 별도 표시

사용자가 수정을 요청하면 반영 후 다시 계획을 보여준다.

### Step 5: 카테고리 페이지 생성

확인 후, 카테고리 페이지를 **깊이 순서대로(위에서 아래로)** 생성한다.

1. 먼저 대분류 카테고리를 상위 페이지 아래에 생성
2. 생성된 대분류의 ID를 사용하여 중분류 카테고리 생성
3. 필요하면 소분류도 생성

각 카테고리에 주제에 맞는 이모지 아이콘을 자동 부여한다.

### Step 6: 페이지 이동

`notion-move-pages`로 하위 페이지들을 각 카테고리로 이동한다.

- 같은 카테고리에 속하는 페이지들은 한 번의 호출로 일괄 이동
- 3단계 구조의 경우, 최종 목적지(가장 깊은 카테고리)로 직접 이동

### Step 7: 결과 보고

완료 후 요약:
- 새로 생성한 카테고리 수
- 이동한 페이지 수
- 최종 트리 구조
- 실패한 항목이 있으면 안내

## 엣지 케이스

### 이미 카테고리가 일부 존재하는 경우
- 기존 카테고리 폴더는 유지하고, 미분류 페이지만 이동 대상으로 삼는다.
- 기존 카테고리에 맞는 미분류 페이지는 해당 카테고리로 이동.

### 데이터베이스가 섞여 있는 경우
- 데이터베이스는 이동 대상에서 제외하고 사용자에게 별도 안내.

### 분류가 애매한 페이지
- 가장 가까운 카테고리에 배치하되, Step 4에서 `(?)` 표시로 사용자에게 판단을 맡긴다.

### 페이지 수가 매우 많은 경우 (50개 이상)
- 배치로 나눠서 처리. `notion-move-pages`의 한 번 호출당 최대 100개 제한 유의.

---

## 주의사항

- **되돌리기 어려움**: `notion-move-pages`는 되돌리기가 어렵다. Step 4 확인을 절대 건너뛰지 않는다.
- **내용 보존**: 이동은 부모만 바꾸며, 페이지 내용은 변경되지 않는다.
- **권한**: 공유 페이지의 경우 이동 권한이 없을 수 있다. 오류 시 안내한다.
